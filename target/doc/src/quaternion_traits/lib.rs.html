2/*!
3This library provides traits and many function for dealing with quaternions.
4
5# Reason
6
7This library exists to provide two things:
8- A way to tie in the multiple existing implementations of
9quaternions in rust.
10- Add functions for pure maths purpaces in rust (like [`sqrt`], [`cos`] and [`pow`](pow_f))
11
12Curently this library is implemented for:
13- [core](https://doc.rust-lang.org/core/)
14- [std](https://doc.rust-lang.org/std/) (feature `std`)
15- [quaternion](https://crates.io/crates/quaternion)
16- [num-complex](https://crates.io/crates/num-complex) (feature `num-complex`)
17- [num-traits](https://crates.io/crates/num-traits) (feature `num-traits`)
18
19# Details
20
21Currently the quaternion traits are implemented only in core rust and num,
22but it's planned to add (though optional dependencies) these traits to crates like
23[bevy](https://crates.io/crates/bevy), [ggez](https://crates.io/crates/ggez)
24and the other quaternion crates out there.
25
26Due to how the traits are implemented this crate is naturaly usable with the
27[quaternion](https://crates.io/crates/quaternion) crate. So for any crates that use
28this crate this dependency is (hopefully) frictionless.
29
30If possible this crate should implement functions for every quaternion use.
31And it should implement every function that a crate it's comapatble with has.
32
33This crate also currently has these pure maths functions for quaternions (excluding common ones):
34`exp`, `ln`, `sqrt`, `sin`, `sinh`, `sec`, `cos`, `cosh`, `csc`, `sin_cos`, `tan`, `tanh`,
35`cot`, `coth`.
36
37This crate provides currently an unstable form of these functions:
38- `pow` (the equasion used seams to not be fully agreed on though so it's at risk of change if
39  another equasion comes out that is guaranteed to be correct)
40- `log` (simple `ln` division only works for complex quaternions that include the real axis and
41  only one of the other 3 imaginary axis. Will remain unstable untill)
42
43List of features:
44- `unstable`: Enables items that may change functionality or may be removed entirely.
45- `std`: Adds `alloc` feature, adds [Std] struct.
46- `alloc`: Adds [Quaternion], [Vector], [Complex] and [Scalar] implementations for
47Box, Arc, Rc and Cow.
48- `num-traits`: (If `unstable` is enabled) adds [Pow](https://docs.rs/num-traits/latest/num_traits/pow/trait.Pow.html) to [Quat],
49(if `std` is enabled) adds [Float](https://docs.rs/num-traits/latest/num_traits/float/trait.Float.html)
50and all the required traits to the [Std] struct.
51- `num-complex`: Adds `num-traits` feature, adds [Complex] implementation for the Complex struct in this crate.
52
53 */
